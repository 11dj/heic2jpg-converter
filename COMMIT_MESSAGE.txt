Title:
------
Fix PowerShell format string error in Windows thumbnail generation

Body:
-----
Fixed the "invalid reference to positional arguments" error in Windows build.

Problem:
- The PowerShell script in generate_thumbnail_base64 used curly braces {}
- These conflicted with Rust's format!() macro syntax
- The format string expected 4 arguments but only 3 were provided
- PowerShell's hashtable syntax @{...} was being interpreted as format arguments

Solution:
- Simplified the PowerShell resize script
- Used [math]::Min() for ratio calculation instead of if-else
- Properly escaped all format arguments
- Single pass with all variables properly formatted

Changed:
- Windows thumbnail generation now uses cleaner PowerShell script
- Calculates ratio using: [math]::Min(max_w/width, max_h/height)
- Creates new bitmap with calculated dimensions
- Exports to PNG and returns base64

Files Changed:
- src-tauri/src/lib.rs (fixed PowerShell script formatting)
